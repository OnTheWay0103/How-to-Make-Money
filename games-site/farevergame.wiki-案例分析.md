# farevergame.wiki 案例分析

> 分析日期：2026-06-28

---

## 一、这个网站是什么

`farevergame.wiki` 是一个**单游戏攻略站**，只做一款游戏——《Farever》（Shiro Games出品的合作动作RPG，2026年5月7日在Steam上架EA版本，售价$17.99）。

它的本质是：**一个人用程序化方式搭建的Wiki，靠AdSense广告赚钱。** 网站明确标注"Unofficial"（非官方），与游戏开发商无关。

**为什么值得研究：** 这是一个教科书级的"发现一个信息缺口 → 快速建站铺内容 → 抢占搜索排名 → 广告变现"的案例。从选品到技术到内容到SEO，打法清晰可复制。

## 三、网站结构（110页怎么组织）

这个站的内容组织遵循 SEO 里常见的 **「Hub页 + Spoke页」** 模式：

- **Hub页**（目录/总览页）：如 `/classes/` 列出所有职业
- **Spoke页**（具体内容页）：如 `/classes/farever-mage-guide/` 单讲法师

完整的页面体系：

```
首页
├── 职业（Classes）
│   ├── 职业总览 / 最佳职业 / 角色创建 / 技能树 / 天赋
│   └── 法师 / 牧师 / 圣骑士 / 盗贼 / 祭司 各职业专题
│
├── 武器（Weapons）
│   ├── 全武器列表 / 双手剑 / 武器技能 / 装备升级
│
├── 流程攻略（Walkthroughs）
│   ├── 新手入门 / 最佳开局 / 第一个副本 / Boss攻略
│   ├── 快速旅行 / 坐骑 / 宠物 / 同伴 / 隐藏生物
│   ├── 制作 / 拆解 / 最大等级 / 合作指南
│
├── 副本（Dungeons）
│   └── 8个副本各自独立页面
│
├── 专题指南（Guides）
│   ├── 最佳职业 / 升级 / 制作 / 坐骑 / 世界观
│   ├── Farever vs 暗黑4 / vs 失落方舟 / vs Wartales（对比测评，捕获搜索流量）
│   ├── 值得买吗 / 单人vs合作
│
├── Tier List（排行榜/强度榜）
├── Server Status（服务器状态）
├── Steam Charts（玩家人数数据）
├── FAQ
├── 发行信息（Release）
│   └── 发售日 / EA / 价格 / PS5 / Xbox / Steam Deck / 成就 / 补丁
├── 多人模式（Multiplayer）
│   └── 合作 / 在线 / 匹配 / 副本 / 组队 / PvP / 社交
└── 多语言版本（/de/ 德语、/es/ 西班牙语、/fr/ 法语）
```

**URL命名规律**：几乎全部是 `farever-xxx` 格式（如 `farever-mage-guide`、`farever-best-class`），把核心搜索词直接嵌在URL里。

## 四、内容是怎么写的

以一个具体页面为例 —— `/guides/farever-best-class/`（最佳职业排行）：

**页面结构：**
```
标题：Farever Best Class — Tier List for Solo & Co-op (2026)
简介：2句话说清本文覆盖什么、基于当前EA版本
正文：
  ├── S-Tier — Ranger（单人模式最强）
  │    理由：弓+陷阱+宠物，生存力高，上手平滑
  ├── S-Tier — Warrior（双人模式最强）
  │    理由：有队友拉仇恨时，仇恨和眩晕价值极大
  ├── A-Tier — Mage（三人组队最强输出）
  │    理由：伤害上限最高但操作门槛也最高
  ├── A-Tier — Mystic（团队辅助核心）
  │    理由：纯辅助过剩，多数人会混搭输出武器
  └── 总结：新手推荐顺序 + 组队推荐顺序
```

**可以看出几个特点：**

1. **简洁直接**：每个评级后1-2句理由，不写长篇大论
2. **分场景**：solo/duo/group 三种场景分别排名，覆盖不同搜索意图
3. **有具体的游戏内容**：提到了Skyover、Autumn等实际地名，说明写手至少了解游戏（不是纯AI瞎编）
4. **版本意识**：标注"当前EA版本"，但**没写具体日期**（这是可以改进的点）

**内容来源推测**：结合Steam页面资料 + 社区讨论（Reddit/Discord）+ 游戏基础体验。不需要"每个职业玩100小时"才能写。

## 五、SEO手段拆解

### 1. 结构化数据（Schema.org）

首页嵌入三种 Schema 标记，帮搜索引擎更好理解这个网站：

- **WebSite + SearchAction**：告诉Google这是个网站，且支持站内搜索
- **VideoGame**：告诉Google这个网站"about"哪款游戏，建立语义关联
- **FAQPage**：首页FAQ直接以富摘要形式出现在搜索结果里（带折叠箭头），**多占一行甚至两行的展示面积**

FAQ页面的7个问题精准命中真实搜索："玩Farever要钱吗""它是MMO吗""服务器挂了吗""什么时候正式上线"——这些都是新玩家决定买不买之前一定会搜的问题。

### 2. 域名自带权重

`farevergame.wiki` — 域名本身包含 "farever game wiki" 这个核心搜索词。虽然现代Google把域名关键词的权重降低了，但在新站初期仍然有帮助。

### 3. URL结构

```
/farever-best-class/     而不是 /guides/best-class/
/farever-mage-guide/     而不是 /classes/mage/
/farever-vs-diablo-4/    精准命中对比评测搜索
```

每条URL都在帮Google理解"这个页面是关于什么的"。

### 4. 多语言扩展

做了德语（7页）、西班牙语（4页）、法语（2页）。德语玩家在Steam上体量很大且付费意愿强，德语游戏攻略内容的竞争程度远低于英语。

## 六、变现分析

### 当前方式：Google AdSense

页面内嵌AdSense广告单元。策略偏克制——没有弹窗、没有打断阅读的插屏广告。对于新站这是对的：先把用户和排名做起来，广告密度可以后加。

### 收入估算

保守假设（游戏EA期，玩家基数小）：

| 指标 | 估算 |
|------|------|
| 日均UV | 2000-8000（取决于搜索排名） |
| 日均PV | 6000-20000 |
| RPM（千次展示收入） | $2-8（游戏类广告偏低） |
| 月收入 | **$150-600** |

乐观假设（如果游戏1.0正式发售爆火）：

| 指标 | 估算 |
|------|------|
| 日均UV | 20000+ |
| 月收入 | **$2000-5000+** |

### 这个站还可以怎么赚钱（目前没做的）

- 联盟链接：Steam游戏推荐可获得分佣
- 游戏推广合作：如果Farever有其他生态产品（周边、DLC），可接CPS推广
- 会员去广告：参考Icy Veins的€0.99/月模式

## 七、这个站做对了什么

1. **选品精准**：选了一款有搜索需求、官方没做Wiki、大站还没覆盖的新游戏。Shiro Games是中型团队，不会像暴雪/任天堂那样有完整的官方攻略体系。

2. **域名即流量**：`farevergame.wiki` 自带关键词。

3. **覆盖面全**：110+页覆盖了一个玩家可能搜索的几乎所有问题。搜索引擎喜欢"结构完整的站"甚于"有几篇深度好文但其他方面空白的站"。

4. **Schema是杀手锏**：FAQPage让搜索结果的展示面积翻倍，CTR天然更高。

5. **上线时间完美**：和游戏EA几乎同步，抢到"新游戏内容缺口"的先发优势。

6. **多语言布局早**：德语已经开工，提前卡位非英语搜索市场。

7. **法律风险规避**：明确标注Unofficial，不与官方混淆。

---

## 八、可以改进的地方

1. **没有计算器/工具型页面**：如果能做一个"Build DPS计算器"或"配装模拟器"，会是完全差异化的高价值内容——Google AI摘要截不走，大站Wiki通常不做。

2. **Tier List缺少数据支撑**：如果能加上社区投票数据或实测数值表，信息密度和可信度会翻倍。

3. **内容深度中等**：目前是"够用"级别。如果后来有更懂游戏的人跟进做更深入的攻略，存在被反超的风险。

4. **og:url处理粗糙**：所有页面og:url都指向首页，社交媒体分享时卡片展示会受影响。

5. **关键词meta全站统一**：所有页面用同一套keywords，每页其实应该用不同的。

6. **版本标注不够精确**：只写了"当前版本"而没有具体补丁号/日期。

## 九、如果要模仿，路径是什么

对于一个不熟悉游戏的人，模仿这个站的关键不在"玩得好"，在**信息组织**：

1. **选游戏**：找一款3个月内有搜索需求的游戏（Steam新品节、EA上线、大版本更新）。标准是：游戏上线几天内就会有人搜"XX攻略""XX值得买吗""XX职业推荐"。

2. **注册域名**：`{游戏名}game.wiki` 或 `{游戏名}wiki.com`

3. **搭网站**：Next.js（这个站用的）或 Astro + Vercel/Cloudflare Pages 免费部署

4. **铺内容**：不要追求每篇精雕细琢。先铺出50-100页的基本覆盖——职业列表、武器列表、新手攻略、FAQ、发售信息。内容来源：Steam页面 + 官方公告 + Reddit讨论 + Discord社区

5. **做Schema**：FAQPage 和 VideoGame 标记一定要上

6. **等Google收录**：通常2-4周开始有展现。用Google Search Console看哪个词有展现没点击，针对性优化

7. **看数据迭代**：哪些页面有流量就深入写，哪些没流量就放一放

## 十、一个更根本的认知

`farevergame.wiki` 的本质不是"一个热爱Farever的玩家做的攻略站"。

它是一个**信息套利机器**：游戏开发商创造了产品（Farever），但不会自己做一个详细的攻略Wiki。Reddit和Discord上有大量零散的玩家讨论，但搜索引擎无法把这些碎片信息整理成结构化的答案。而Google仍然需要把搜索用户送到某个网页——这个机会被这个站抓住了。

对于不熟悉游戏的人来说，你的价值不在"玩游戏"，在**把散落在互联网各处但没人整理成网页的信息，整理成搜索引擎喜欢的格式**。
