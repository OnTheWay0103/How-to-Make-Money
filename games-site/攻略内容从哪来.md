# 攻略内容从哪来

> 两个核心问题：玩家为什么需要外部攻略网站？不亲自玩游戏能不能做攻略站？

---

## 第一部分：玩家为什么需要外部攻略网站

### 游戏里明明什么都有，为什么还要搜攻略？

答案很简单：**游戏只告诉你"这是什么"，不告诉你"这对你来说意味着什么"。**

---

### 一、游戏里的信息 vs 玩家真正需要的信息

拿一个典型RPG来说。你在游戏里看到的装备对比界面：

```
装备A：攻击力 +45，暴击率 +12%
装备B：攻击力 +38，暴击伤害 +35%
```

游戏只给你这两行数字。现在你要决定穿哪件。

但你**真正需要知道**的却是另一套东西——实际伤害公式（以《原神》为例，这是社区逆向推导出来的）：

```
实际伤害 = 基础攻击力 × (1 + 攻击力加成%) × 技能倍率
          × (1 + 元素伤害加成%) × (1 + 暴击伤害%)
          × 敌人抗性系数 × 敌人防御系数 × 元素反应系数
```

**游戏里只显示各项"面板数字"，但最终伤害取决于七八个变量的乘积。** 人脑不可能心算这堆乘法，而游戏又不提供"穿上装备后对XX敌人的期望伤害预览"。

这就是外部攻略站存在的第一个原因：**公式被藏起来了，需要有人逆向推导并做成工具。**

---

### 二、更关键的因素：资源投入不可逆

这才是让玩家不敢凭感觉做决定的根本原因：

| 游戏 | 一次错误选择的代价 |
|------|-------------------|
| 《原神》 | 升级一个圣遗物到满级需要刷几十次副本的材料。升错了=浪费几天时间+宝贵的体力 |
| 《艾尔登法环》 | 洗点道具极其有限（一周目可能只有十几个）。点错只能下周目重来 |
| 《暗黑破坏神4》 | 洗巅峰盘的金币成本指数增长，后期洗一次几百万 |
| 《崩坏：星穹铁道》 | 周本Boss材料一周只能打3次，用错角色=浪费一周进度 |

玩家在NGA、Reddit上搜"该升哪件""该抽哪个"，搜的不是攻略，是**在投入不可逆资源之前的决策信心**。

外部工具解决的问题是：

```
我有5件圣遗物 → 计算器跑一遍 → "方案C的期望伤害最高，比方案B高8%"
```

这个结果，游戏界面绝对不会告诉你。

---

### 三、计算器型页面的价值（为什么CTR比纯文字攻略高3-5倍）

之前提到Roblox攻略站开发者发现计算器/排行榜页面的点击率比纯文字攻略高3-5倍。这里具体解释为什么。

#### 场景1：配装选择

**游戏里看到的：**
```
武器A：基础攻击510 + 暴击率27.6%
武器B：基础攻击674 + 元素充能45.1%
```

**看似B攻击高，但实际答案取决于：**
- 攻击白值高就一定好吗？不——它乘的是你当前攻击力加成的总和。加成低的时候，高白值的收益反而打折扣
- 暴击率和充能怎么量化比较？取决于你当前的暴击伤害面板和队伍充能需求
- 结论因你的队伍、圣遗物、甚至打的敌人而异

**计算器做的事：** 你把当前面板输进去，它完整跑一遍伤害公式，告诉你"对你当前配置，武器A期望伤害高6.3%"。两个数字的人脑比较 → 七八个变量的乘积比较，游戏和计算器之间隔了本质的距离。

#### 场景2：队伍搭配（游戏根本帮不了你）

一个四人队伍，角色池里有20个可选角色。理论组合数是 C(20,4) × 出场顺序排列，不可能一个个试。外部计算器做的是：输入敌人属性 → 模拟不同配队的每秒伤害 → 排名。

#### 场景3：材料规划

"我要把角色从1级拉到90级，天赋全满"。游戏里只显示"下一步缺什么"，但**不告诉你总需求**：

```
摩拉总计：705万
经验书：419本
天赋材料：9绿 + 63蓝 + 114紫
Boss掉落物：46个
突破石头：碎屑×1 + 断片×9 + 块×9 + 完整×6
预估所需天数：12天（按每日体力上限计算）
```

一个材料计算器5秒钟告诉你结果和需要的天数。用游戏内界面查，你得来回切十几个页面自己做加法。

---

### 四、为什么大站Wiki普遍不做这些

既然需求这么大，为什么Fandom Wiki这类最大的游戏Wiki通常不提供计算器？

| 原因 | 说明 |
|------|------|
| **Wiki是静态内容系统** | 设计上就是编辑文本、上传图片。计算器需要前端交互逻辑和公式引擎，和Wiki的技术架构不兼容 |
| **志愿者编辑不会写代码** | Wiki编辑擅长写文字攻略，但不会写JavaScript来模拟循环DPS |
| **公式随版本更新变化** | 游戏每次更新调数值，计算器都要跟着改代码。纯志愿者模式无法保证维护速度 |
| **Wiki的商业模式不激励开发工具** | 靠捐款和广告维持，没有动力投入开发资源 |

这恰好是一个**结构性机会**：大站做了攻略文字部分，但计算器这个高价值组件是缺失的。独立站做一个轻量计算器，就能在搜索结果里差异化——因为Google没法用AI摘要替代一个可交互的计算器。

---

### 五、通俗类比

把游戏攻略站类比为汽车评测会更直观：

| | 游戏 | 汽车 |
|------|------|------|
| **游戏内信息** | 面板数字、技能描述 | 4S店看到的：外观、标称油耗、官方0-100 |
| **攻略文字** | Boss打法、配装推荐 | 车评人试驾体验："变速箱调校偏舒适" |
| **计算器** | 输入你的面板，算最优配装 | 输入预算、通勤距离、充电条件 → "Model Y比极氪001五年总成本低2.8万" |

玩家要的不是"知道数据"，而是**基于自己的具体情况做出最优决策**。计算器解决的是最后一公里：把通用信息变成个性化答案。

## 第二部分：攻略一定要亲自玩才能写吗

### 直接回答

**不需要全部自己玩。** 大部分攻略站的内容来源比你想象的更"组合"。

---

### 攻略的六种来源

#### 1. 亲自玩（约占10-20%）

这是最扎实但最慢的方式。适合的情况：
- 写你最熟悉的1-2款游戏
- "手感向"内容——比如"这个Boss P2阶段往右翻滚比往左安全"，这类体感经验确实只能自己打出来
- 需要对比"社区说好"和"实际好用"的时候

但如果每篇攻略都靠亲自打通，一个3A游戏动辄50-100小时，一个月写不了几篇。

#### 2. 数据挖掘（约占20-30%）

这是硬核攻略站最重要的来源，**不需要玩，需要解包**。

怎么做：游戏客户端更新时，用解包工具提取配置文件。伤害公式、掉落率、装备属性、角色成长曲线——这些不是"玩出来"的，是**从代码和配置表里扒出来的**。

例如《原神》的伤害公式、圣遗物掉落权重、卡池概率分布，最早都是数据挖掘者公布到社区，不是谁"测试"出来的。Icy Veins的魔兽攻略里大量精确数值，数据来源也是暴雪公开API和社区解包数据。

工具取决于游戏引擎：Unity游戏用AssetStudio，Unreal Engine游戏用FModel，有公开API的游戏直接拉接口。

#### 3. 社区整理（约占30-40%）⭐ 最高效

这是**最高效且完全合法**的方式。你不是在创造信息，你是在**结构化散落在社区里的信息**。

真实工作流：

```
打开 Reddit / NGA / Discord / B站评论区
  → 搜"XX Boss 打不过""XX角色怎么配"
    → 看高赞回答里已经经过社区群众投票验证的最优解
      → 你做的：把零散讨论整理成结构化的攻略页面
```

**具体例子：** 你要写《艾尔登法环》玛莲妮亚的攻略。

- 你不需要自己死200次。Reddit上已经有几千条打法讨论，B站上有几十个教学视频
- 你要做的：把"出血流打法"从5个帖子里提炼成一套分阶段操作指引 + 配装表 + 错误示范
- 你的价值不是"发现信息"，是**把信息整理成玩家30秒能看完的东西**

这是大部分职业攻略写手的真实工作方式。Icy Veins的写手虽然本身是高手玩家，但他们产出的大部分内容也是在自己玩的基础上结合社区验证，而不是闭门从头测起。

#### 4. 翻译搬运（约占10-20%）

日本攻略站、韩国论坛、英文Reddit——很多信息在不同语言圈子里是隔离的。

一个韩服的新Build可能要过几周才会被中文社区注意到。如果你会外语，**翻译+本地化**就是极高的信息差套利。Game8早期的很多内容就是搬运+整理日服玩家在推特和5ch的讨论。

#### 5. 官方资料（约占5-10%）

版本更新公告、开发者博客、官方Wiki。这些是公开的、官方的、不需要玩游戏就能获取的。

#### 6. AI辅助（新兴方式）

在以上五种来源的基础上，AI可以用来：
- 把社区讨论提炼成结构化大纲
- 把视频攻略转成文字版分步骤指南
- 生成多语言版本
- 从解包数据自动生成装备对比表

**但AI不能替代1-5的任何一步**。它没有"体感"，不知道哪个打法更好用，也不能帮你验证社区信息的真伪。AI的角色是"加速器"而非"信息源"。

---

### 各种来源的对比

```
攻略来源        需要玩游戏吗     产出速度    信息可信度    适合什么
─────────────────────────────────────────────────────────
亲自玩           ✅ 需要          最慢        最高          你最熟悉的游戏
数据挖掘         ❌ 不需要        快          最高          数值不会骗人
社区整理         ❌ 不需要        最快        中高          主力生产方式
翻译搬运         ❌ 不需要        快          中            跨语言信息差
官方资料         ❌ 不需要        快          最高          基础信息
AI辅助           ❌ 不需要        辅助用      需人工验证    加速器角色
```

---

### 真正重要的问题不是"要不要自己玩"，而是"玩到什么程度"

你需要玩到**能判断信息质量**的程度。

不用亲自用每种Build打通游戏，但需要玩过这个游戏，能判断：
- 社区推荐的"T0配装"实际手感是不是真的适合大多数玩家？
- Reddit高赞回复是硬核玩家的偏好，还是对普通玩家也适用？
- 某攻略说"简单"的操作，对手残党来说是不是地狱难度？

**玩100小时是必须的吗？不是。但完全不玩、纯搬运，会被玩家一眼看穿。**

---

### 最佳状态

主攻1-2款你真正在玩的游戏（亲自写深度内容），其余游戏靠"数据挖掘 + 社区整理 + 翻译搬运"组合扩大覆盖。

这就是 Icy Veins 的扩张模型——先从魔兽这个自己擅长的游戏做透，再招其他游戏的专家写手用同样的方法论覆盖新游戏。

---

### 对不熟悉游戏的人的三个可行路径

如果你本身对游戏不熟，以下是三条实际可走的路：

#### 路径A：纯SEO驱动（不依赖游戏知识）

1. 通过Google Trends和关键词工具找到"搜索量大但竞争小"的游戏
2. 从Steam页面、官方公告、Reddit高赞帖里提取信息
3. 用模板化方式写成结构化的攻略页面
4. 不需要深度游戏理解也能做：FAQ页、发行信息页、系统需求页、Steam数据页

#### 路径B：工具型网站（不写攻略，做计算器）

1. 找一个游戏，从社区获取其核心公式（伤害公式、掉落公式等）
2. 用JavaScript写成交互式计算器
3. 这种内容天然比文字攻略更难被AI截流，也更难被搬运

#### 路径C：合作模式

1. 找一个懂游戏的人负责内容方向和质量把控
2. 你负责网站搭建、SEO、变现、运营
3. 这是很多成功的游戏攻略站背后的真实分工——Icy Veins的创始人之一负责技术和商业化，写手团队负责内容
