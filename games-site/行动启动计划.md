# 游戏攻略站 — 行动启动计划 v1

> 2026-06-29 | 基于九份文档的交叉分析，提炼为可执行的行动方案。

---

## 一、总体策略

**先走路径A试水（跟新游戏），再切路径B铺量（老游戏长尾）。**

理由：
- 路径A两周可验证（网站有没有被收录、有没有展现），试错成本低
- 路径A只需30-50页，路径B需要200-500页
- 跑通一遍流程后，切路径B时已经知道怎么写、怎么铺、怎么等收录

**核心原则：MVP不是"系统"，是"30篇能进Google的攻略页"。**

## 二、选游戏：从哪找、怎么筛

### 数据源

| 来源 | 链接 | 用途 |
|------|------|------|
| Steam 新发售 | store.steampowered.com/explore/new/ | 找已上线的新游戏 |
| Steam 即将推出 | store.steampowered.com/explore/upcoming/ | 提前布局未上线游戏 |
| SteamDB | steamdb.info/upcoming/ | 看更全的即将上线列表 |
| Steam 新品节 | 每年2/6/10月 | 集中爆发期，一次几十款新游 |

### 筛选标准（五条，缺一条不碰）

1. **开发商规模中等**（20-100人团队）：不会像暴雪有官方攻略站，但游戏本身有玩家基数
2. **RPG/生存/动作类优先**：有职业、装备、Build、Boss → 能拆出大量攻略页
3. **Steam评价≥70%好评**：太差没人玩，攻略做了没流量
4. **上线后7天内搜索量明显上升**（用Google Trends验证）
5. **大站还没做专题**：搜"游戏名 wiki"前五没有 Fandom/IGN/GamesRadar 的专题页

## 三、网站结构（极简版）

### URL规划

```
首页 /
├── /guides/           ← hub页：攻略列表
│   ├── /guides/[游戏名]-beginner-guide/
│   ├── /guides/[游戏名]-best-build/
│   ├── /guides/[游戏名]-boss-guide/
│   └── ...
├── /tier-list/        ← 强度排行（高CTR页面）
├── /release/          ← 发行信息（发售日/价格/平台/配置）
├── /faq/              ← FAQ（Schema标记）
└── /comparison/       ← 对比评测（如"XX vs YY"）
```

### 每页必备元素

- 适用版本标注 + 更新日期
- FAQ区块（用FAQPage Schema）
- 相关页面内链（3-5条）
- 移动端优先设计

## 四、内容生产流程

### 单篇攻略的信息来源

| 优先级 | 来源 | 具体操作 |
|--------|------|---------|
| 1 | Steam商店页 | 获取游戏基础信息、配置要求、官方描述 |
| 2 | 官方更新公告/开发者博客 | 版本机制、数值变化 |
| 3 | Reddit（r/游戏名） | 玩家高频问题、高赞打法 |
| 4 | YouTube攻略视频标题 | 提取关键词、了解玩家卡在哪 |
| 5 | Discord社区 | 直接看到玩家在问什么 |

### 写作模板

```markdown
# [游戏名] [内容主题] — Guide (v[版本号])

**Last updated:** [日期] | **适用于:** [版本号]

## Quick Answer（前100字直接回答搜索意图）

## Step-by-Step（分步骤详细说明）

## Common Mistakes（常见错误，来自Reddit/Discord踩坑帖）

## FAQ（3-5个相关问题）

## Related Guides（3-5条内链）
```

### 节奏

- 第1周：10篇核心攻略（新手入门、最佳职业、首个Boss、FAQ、发行信息）
- 第2周：+15篇扩展攻略（职业专题、武器列表、更多Boss）
- 第3-4周：按Search Console数据补充有展现缺点击的页面

## 五、技术方案

### MVP推荐方案：Next.js + Vercel 免费部署

```
框架：Next.js App Router
内容：Markdown/MDX（直接写在代码仓库里，省去数据库）
样式：Tailwind CSS
部署：Vercel（免费额度：100GB带宽/月，够初期用）
域名：Namesilo / Namecheap 购买（$5-15/年）
分析：Google Search Console + Google Analytics + Vercel Analytics
广告：Google AdSense（等有流量了再加，不要一上来就放广告）
```

### 为什么不用WordPress

对于"选新游戏 → 快速铺模板化内容"这个场景：
- Next.js 的页面生成更灵活，URL结构完全可控
- Markdown写内容比WordPress编辑器更高效
- Vercel部署零运维
- farevergame.wiki 也是这个技术栈，已验证可行

## 八、砍掉不做的事（MVP阶段）

- ❌ 不做多游戏平台（一个游戏一个站，做好了再复制）
- ❌ 不做自动化爬虫/关键词系统（先手动跑通流程）
- ❌ 不做用户注册/评论系统（社区互动是负担，先专注内容）
- ❌ 不做复杂的数据库/后端（Markdown + Git 就是CMS）
- ❌ 不买付费SEO工具（Google Suggest + Search Console 免费够用）
- ❌ 不做视频内容（先专注文字+截图，视频是下一个阶段的事）

## 九、实操指南1：如何确认 Steam 当前/即将上线的游戏

### 核心数据源（每天15分钟）

| 序号 | 来源 | 链接 | 用途 |
|------|------|------|------|
| 1 | Steam 热门新游 | store.steampowered.com/search/?filter=popularnew&sort_by=Released_DESC | 找已上线、玩家已经在买的游戏 |
| 2 | Steam 热门即将推出 | store.steampowered.com/explore/upcoming/ | 愿望单排名最高的未上线游戏，预判爆款 |
| 3 | SteamDB | steamdb.info/upcoming/ | 比官方多了 Followers 数据、精确发售日 |
| 4 | Steam 新品节 | 每年2/6/10月，steamnextfest.com | 一年三次，几百款游戏集中放试玩Demo |

### 筛选流程（每次15分钟）

```
打开 Steam 新发售页面
  ↓
只看 RPG / 动作 / 生存 / 合作 这四类
  ↓
跳过明显独立小作品（截图看起来像一个人做的、评价数<50）
  ↓
跳过3A大作（Fandom Wiki会瞬间覆盖，你没机会）
  ↓
剩下的游戏 → 打开 Google Trends，输入游戏名
  ↓
搜索量在上升 → 搜"游戏名 wiki"看前五有没有大站专题
  ↓
没有 → 候选目标 ✓
```

### 快速判断技巧

一款游戏上线3天内 Steam 评价数超过200条 → 销量已经不差 → 搜索量大概率在爬升 → 值得做。

## 十、实操指南2：从哪里获取用户搜索的关键词

### 免费方法（MVP阶段完全够用）

**方法1：Google 搜索建议（最核心、全免费）**

在 Google 搜索框输入游戏名 + 前缀，自动补全 = 真实玩家搜索：

```
游戏名 + how        → how to beat / how to level up / how to get
游戏名 + best       → best weapon / best build / best class
游戏名 + guide      → beginner guide / complete guide
游戏名 + fix        → fix crash / fix fps / fix lag
游戏名 + vs         → XX vs YY（对比评测）
游戏名 + is         → is XX worth it / is XX multiplayer
```

**方法2：Google "People also ask"（长尾问题金矿）**

搜索任意一个攻略词 → 结果页中间的"用户还问了以下问题"区域 → 每个问题 = 一篇攻略标题。点击展开一个问题后会追加2-3个新问题，一层层点下去能挖出15-20个长尾词。

**方法3：Reddit**

```
reddit.com/r/[游戏名] → Top → Past Month
```

玩家在这里用最自然的语言问问题。这些问题翻译成SEO标题就是攻略选题。

**方法4：YouTube 搜索建议**

同Google Suggest逻辑，但YouTube建议词往往更具体、更口语化，而这些词在文字搜索里竞争可能还很低。

**方法5：AnswerThePublic**

```
answerthepublic.com
```

免费版每天几次查询，输入游戏名生成"问题地图"（what/how/why/when/where/which）。

**方法6：Google Trends（验证趋势，不是挖词）**

```
trends.google.com
```

用途：验证你找到的词值不值得做——看搜索量趋势是上升还是下降、对比候选游戏、看欧美流量占比（决定AdSense RPM）。

### 付费工具（铺量阶段再买）

| 工具 | 月费 | 用途 |
|------|------|------|
| Ahrefs | $129/月 | 批量分析竞品关键词 |
| SEMrush | $139/月 | 同上 |
| Keywords Everywhere | $15/年 | 浏览器插件，直接显示搜索量 |

### 一个游戏的完整挖词流程（约30-40分钟）

```
Step 1：Google搜索"XXX how" → 记下自动补全10个词
Step 2：Google搜索"XXX best" → 记下自动补全10个词
Step 3：每个词点进搜索结果 → 收集"People also ask"的5-8个问题
Step 4：Reddit r/XXX → Top posts → 提取玩家问题
Step 5：YouTube搜索"XXX guide" → 记下高播放量视频标题
Step 6：所有词扔进Google Trends → 验证趋势 → 按潜力排序
Step 7：得到50-80个关键词 → 这就是前两周内容计划
```
