# 一、我是怎么筛选“能赚钱的游戏”的（核心方法）

我不是用“热度”选，而是用 4 个维度筛：

---

## ✅ 维度1：搜索需求结构（最重要）

一个游戏能不能赚钱，看它有没有这些关键词结构：

### 🔥 高价值关键词类型：

* how to / beginner guide
* best build / tier list
* settings / optimization
* fix / crash / error
* patch notes / update

---

👉 如果一个游戏能生成 ≥ 200 个这种词 → 就是“SEO金矿”

---

## ✅ 维度2：生命周期（时间窗口）

分三类：

| 类型         | 特征   | 是否适合  |
| ---------- | ---- | ----- |
| 新发售（0–6个月） | 搜索爆发 | ⭐⭐⭐⭐⭐ |
| 稳定期（1–5年）  | 长尾稳定 | ⭐⭐⭐⭐⭐ |
| 过气游戏       | 搜索下降 | ❌     |

---

## ✅ 维度3：内容可扩展性

我会判断：

> 一个游戏能不能拆出“100–500篇内容”

标准：

* 有职业 / build系统
* 有装备 / 技能树
* 有关卡 / boss
* 有更新patch

---

## ✅ 维度4：竞争结构

优先：

* IGN / Fandom 未完全垄断
* Reddit 活跃但未SEO化
* Steam游戏优先

---

# 二、我是如何判断“用户会搜什么关键词”的

这个是关键能力，我拆给你：

---

## 方法1：Google Suggest（真实用户输入）

输入：

* game name + how
* game name + best
* game name + fix

👉 Google自动补全 = 真实搜索

---

## 方法2：YouTube标题（强信号）

搜索：

> game name gameplay / guide / tips

你会看到：

* “best starter guide”
* “top 10 builds”
* “don’t make this mistake”

👉 YouTube ≈ SEO关键词生成器

---

## 方法3：Reddit问题（最早需求）

看：

* “What should I do first?”
* “Why is my game crashing?”
* “Best class?”

👉 Reddit = 未结构化搜索词源头

---

## 方法4：Steam评论（隐藏关键词）

差评/好评中：

* “too hard”
* “confusing build system”
* “lag issues”

👉 这些会变成 SEO关键词
